# 정렬 알고리즘

## 1. 버블 정렬 (Bubble Sort)

### 정의
인접한 두 원소를 비교해서 순서가 잘못되어 있으면 교환하는 정렬.  
한 사이클이 끝나면 가장 큰 값이 맨 뒤로 "버블처럼" 올라온다.

### 특징
| 항목 | 내용 |
|------|------|
| 시간 복잡도 (평균/최악) | O(n²) |
| 시간 복잡도 (최선) | O(n) — 이미 정렬된 경우 |
| 공간 복잡도 | O(1) |
| 안정 정렬 | O (같은 값의 순서 보장) |

### 동작 과정
```
[5, 3, 1, 4, 2]

1회전: [3,1,4,2, **5**]  ← 5가 맨 뒤로
2회전: [1,3,2, **4**,5]  ← 4가 제자리
3회전: [1,2, **3**,4,5]
4회전: [**1**,2,3,4,5]
```

### Java 구현
```java
void bubbleSort(int[] arr) {
    int n = arr.length;
    for (int i = 0; i < n - 1; i++) {
        boolean swapped = false;
        for (int j = 0; j < n - 1 - i; j++) {
            if (arr[j] > arr[j + 1]) {
                int tmp = arr[j];
                arr[j] = arr[j + 1];
                arr[j + 1] = tmp;
                swapped = true;
            }
        }
        if (!swapped) break;
    }
}
```

# 자료구조

## 2. 배열 (Array)

### 정의
동일한 타입의 원소들을 **연속된 메모리 공간**에 저장하는 자료구조.  
인덱스를 통해 O(1)로 임의 접근 가능.

### 특징
| 연산 | 시간 복잡도 |
|------|------------|
| 접근 (index) | O(1) |
| 탐색 | O(n) |
| 삽입/삭제 (중간) | O(n) |
| 삽입/삭제 (끝) | O(1) |

### Java 예시
```java
int[] arr = {10, 20, 30, 40, 50};

System.out.println(arr[2]);   // 30 — O(1) 접근

for (int i = 0; i < arr.length; i++) {
    if (arr[i] == 30) System.out.println("found at " + i);
}
```

## 3. 연결 리스트 (Linked List)

### 정의
각 노드가 **데이터**와 **다음 노드를 가리키는 포인터(next)**를 가진 선형 자료구조.  
메모리가 연속적이지 않아도 되며, 앞 삽입/삭제가 O(1).

```
[10 | *] → [20 | *] → [30 | null]
```

### 특징
| 연산 | 시간 복잡도 |
|------|------------|
| 접근 | O(n) |
| 탐색 | O(n) |
| 삽입/삭제 (앞) | O(1) |
| 삽입/삭제 (중간/끝) | O(n) — 탐색 포함 |

### 배열 vs 연결 리스트
| | 배열 | 연결 리스트 |
|-|------|------------|
| 메모리 | 연속 | 분산 |
| 접근 | O(1) | O(n) |
| 앞 삽입 | O(n) | O(1) |
| 크기 변경 | 고정 (Java: ArrayList는 동적) | 동적 |

### Java 구현
```java
class Node {
    int data;
    Node next;
    Node(int data) { this.data = data; }
}

class LinkedList {
    Node head;

    void addFirst(int data) {
        Node node = new Node(data);
        node.next = head;
        head = node;
    }

    void addLast(int data) {
        Node node = new Node(data);
        if (head == null) { head = node; return; }
        Node cur = head;
        while (cur.next != null) cur = cur.next;
        cur.next = node;
    }

    void print() {
        Node cur = head;
        while (cur != null) {
            System.out.print(cur.data + " → ");
            cur = cur.next;
        }
        System.out.println("null");
    }
}
```

## 4. 이중 연결 리스트 (Doubly Linked List)

### 정의
각 노드가 **이전 노드(prev)**와 **다음 노드(next)** 포인터를 모두 가진 연결 리스트.  
양방향 탐색이 가능하고, prev 덕분에 특정 노드 삭제가 O(1).

```
null ← [10 | * | *] ↔ [20 | * | *] ↔ [30 | * | *] → null
```

### 단순 연결 리스트와의 차이
| | 단순 | 이중 |
|-|------|------|
| 포인터 수 | 1 (next) | 2 (prev + next) |
| 역방향 탐색 | X | O |
| 노드 삭제 | O(n) — 앞 노드 탐색 필요 | O(1) — prev로 바로 접근 |
| 메모리 | 적음 | 많음 |

### Java 구현
```java
class DNode {
    int data;
    DNode prev, next;
    DNode(int data) { this.data = data; }
}

class DoublyLinkedList {
    DNode head, tail;

    void addLast(int data) {
        DNode node = new DNode(data);
        if (tail == null) { head = tail = node; return; }
        tail.next = node;
        node.prev = tail;
        tail = node;
    }

    void delete(DNode node) {
        if (node.prev != null) node.prev.next = node.next;
        else head = node.next;
        if (node.next != null) node.next.prev = node.prev;
        else tail = node.prev;
    }
}
```

## 5. 원형 연결 리스트 (Circular Linked List)

### 정의
마지막 노드의 next가 **다시 첫 번째 노드(head)를 가리키는** 연결 리스트.  
순환 구조로, 끝이 없어 순환 큐나 라운드 로빈 스케줄링에 활용된다.

```
→ [10] → [20] → [30] ─┐
   ↑___________________|
```

### Java 구현
```java
class CNode {
    int data;
    CNode next;
    CNode(int data) { this.data = data; }
}

class CircularLinkedList {
    CNode head;

    void add(int data) {
        CNode node = new CNode(data);
        if (head == null) { head = node; node.next = head; return; }
        CNode cur = head;
        while (cur.next != head) cur = cur.next;
        cur.next = node;
        node.next = head;
    }

    void print(int count) {
        CNode cur = head;
        for (int i = 0; i < count; i++) {
            System.out.print(cur.data + " → ");
            cur = cur.next;
        }
        System.out.println("(head)");
    }
}
```

## 6. 해시 테이블 (Hash Table)

### 정의
**해시 함수**를 이용해 키(key)를 인덱스로 변환하고, 해당 인덱스에 값을 저장하는 자료구조.  
평균 O(1)로 탐색·삽입·삭제가 가능하다.

### 핵심 개념
- **해시 함수**: key → index 변환 (예: `key.hashCode() % tableSize`)
- **충돌(Collision)**: 서로 다른 키가 같은 인덱스를 가질 때
- **충돌 해결**:
  - **체이닝(Chaining)**: 같은 인덱스에 연결 리스트로 연결
  - **개방 주소법(Open Addressing)**: 빈 슬롯을 탐색해 저장

### 특징
| 연산 | 평균 | 최악 (충돌 많을 때) |
|------|------|---------------------|
| 탐색 | O(1) | O(n) |
| 삽입 | O(1) | O(n) |
| 삭제 | O(1) | O(n) |

### Java 예시 (HashMap)
```java
import java.util.HashMap;

HashMap<String, Integer> map = new HashMap<>();
map.put("apple", 3);
map.put("banana", 5);

System.out.println(map.get("apple"));          // 3
System.out.println(map.containsKey("banana")); // true
map.remove("banana");

// 순회
for (Map.Entry<String, Integer> e : map.entrySet())
    System.out.println(e.getKey() + " = " + e.getValue());
```

## 7. 튜플 & 레코드 (Tuple & Record)

### 정의
- **튜플(Tuple)**: 순서가 있는 **불변(immutable)** 값의 묶음. 각 위치에 의미가 있다.
- **레코드(Record)**: 필드명을 가진 데이터 묶음. Java 16+의 `record` 키워드로 불변 클래스를 간결하게 선언 가능.

### 튜플 vs 레코드 비교
| 항목 | 튜플 | 레코드 |
|------|------|--------|
| 접근 방식 | 인덱스 (0, 1, 2...) | 필드명 |
| 가변성 | 불변 | 불변 |
| 의미 전달 | 낮음 | 높음 |

### Java 예시
```java
// Java의 record (Java 16+)
record Point(int x, int y) {}

Point p = new Point(3, 4);
System.out.println(p.x());  // 3
System.out.println(p.y());  // 4

// 튜플처럼 활용하는 제네릭 Pair
record Pair<A, B>(A first, B second) {}

Pair<String, Integer> pair = new Pair<>("score", 100);
System.out.println(pair.first() + ": " + pair.second()); // score: 100
```

## 자료구조 비교 요약

| 자료구조 | 접근 | 탐색 | 삽입 | 삭제 | 특징 |
|----------|------|------|------|------|------|
| 배열 | O(1) | O(n) | O(n) | O(n) | 인덱스 기반 빠른 접근 |
| 연결 리스트 | O(n) | O(n) | O(1) | O(1) | 동적 크기, 앞 삽입 빠름 |
| 이중 연결 리스트 | O(n) | O(n) | O(1) | O(1) | 양방향 탐색, 노드 삭제 O(1) |
| 원형 연결 리스트 | O(n) | O(n) | O(1) | O(1) | 끝이 없는 순환 구조 |
| 해시 테이블 | - | O(1)* | O(1)* | O(1)* | 키-값 쌍, 충돌 주의 |

> \* 평균 시간 복잡도
