# 정렬 알고리즘

## 1. 버블 정렬 (Bubble Sort)

### 정의
인접한 두 원소를 비교해서 순서가 잘못되어 있으면 교환하는 정렬.  
한 사이클이 끝나면 가장 큰 값이 맨 뒤로 "버블처럼" 올라온다.

### 특징
| 항목 | 내용 |
|------|------|
| 시간 복잡도 (평균/최악) | O(n²) |
| 시간 복잡도 (최선) | O(n) — 이미 정렬된 경우 |
| 공간 복잡도 | O(1) |
| 안정 정렬 | O (같은 값의 순서 보장) |

### 동작 과정
```
[5, 3, 1, 4, 2]

1회전: [3,1,4,2, **5**]  ← 5가 맨 뒤로
2회전: [1,3,2, **4**,5]  ← 4가 제자리
3회전: [1,2, **3**,4,5]
4회전: [**1**,2,3,4,5]
```

### Java 구현
```java
void bubbleSort(int[] arr) {
    int n = arr.length;
    for (int i = 0; i < n - 1; i++) {
        boolean swapped = false;
        for (int j = 0; j < n - 1 - i; j++) {
            if (arr[j] > arr[j + 1]) {
                int tmp = arr[j];
                arr[j] = arr[j + 1];
                arr[j + 1] = tmp;
                swapped = true;
            }
        }
        if (!swapped) break;
    }
}
```

## 2. 선택 정렬 (Selection Sort)

### 정의
전체 배열에서 최솟값을 찾아 맨 앞 원소와 교환하는 작업을 반복하는 정렬.

### 특징
| 항목 | 내용 |
|------|------|
| 시간 복잡도 (평균/최악/최선) | O(n²) |
| 공간 복잡도 | O(1) |
| 안정 정렬 | X (교환 시 순서 뒤바뀔 수 있음) |

### 동작 과정
```
[5, 3, 1, 4, 2]

i=0: 최솟값 1 → swap(5,1) → [1, 3, 5, 4, 2]
i=1: 최솟값 2 → swap(3,2) → [1, 2, 5, 4, 3]
i=2: 최솟값 3 → swap(5,3) → [1, 2, 3, 4, 5]
i=3: 최솟값 4 → 그대로   → [1, 2, 3, 4, 5]
```

### Java 구현
```java
void selectionSort(int[] arr) {
    int n = arr.length;
    for (int i = 0; i < n - 1; i++) {
        int minIdx = i;
        for (int j = i + 1; j < n; j++) {
            if (arr[j] < arr[minIdx]) minIdx = j;
        }
        int tmp = arr[i];
        arr[i] = arr[minIdx];
        arr[minIdx] = tmp;
    }
}
```

## 3. 삽입 정렬 (Insertion Sort)

### 정의
정렬된 부분 배열에 새 원소를 올바른 위치에 삽입하는 방식으로 확장해 나가는 정렬.  
카드 게임에서 패를 정렬하는 방법과 동일한 원리.

### 특징
| 항목 | 내용 |
|------|------|
| 시간 복잡도 (평균/최악) | O(n²) |
| 시간 복잡도 (최선) | O(n) — 이미 정렬된 경우 |
| 공간 복잡도 | O(1) |
| 안정 정렬 | O |

### 동작 과정
```
[5, 3, 1, 4, 2]

i=1: key=3 → [3, 5, 1, 4, 2]
i=2: key=1 → [1, 3, 5, 4, 2]
i=3: key=4 → [1, 3, 4, 5, 2]
i=4: key=2 → [1, 2, 3, 4, 5]
```

### Java 구현
```java
void insertionSort(int[] arr) {
    int n = arr.length;
    for (int i = 1; i < n; i++) {
        int key = arr[i];
        int j = i - 1;
        while (j >= 0 && arr[j] > key) {
            arr[j + 1] = arr[j];
            j--;
        }
        arr[j + 1] = key;
    }
}
```

## 4. 퀵 정렬 (Quick Sort)

### 정의
**피벗(pivot)**을 기준으로 작은 값은 왼쪽, 큰 값은 오른쪽으로 분할한 뒤 각 부분을 재귀적으로 정렬하는 분할 정복 정렬.

### 특징
| 항목 | 내용 |
|------|------|
| 시간 복잡도 (평균) | O(n log n) |
| 시간 복잡도 (최악) | O(n²) — 이미 정렬된 배열에서 첫/끝 피벗 선택 시 |
| 공간 복잡도 | O(log n) — 재귀 스택 |
| 안정 정렬 | X |

### 동작 과정
```
[3, 6, 8, 10, 1, 2, 1]  pivot = 3

왼쪽: [1, 2, 1]  |  pivot: 3  |  오른쪽: [6, 8, 10]

재귀 → [1, 1, 2] + [3] + [6, 8, 10]
결과: [1, 1, 2, 3, 6, 8, 10]
```

### Java 구현
```java
void quickSort(int[] arr, int low, int high) {
    if (low < high) {
        int pi = partition(arr, low, high);
        quickSort(arr, low, pi - 1);
        quickSort(arr, pi + 1, high);
    }
}

int partition(int[] arr, int low, int high) {
    int pivot = arr[high];
    int i = low - 1;
    for (int j = low; j < high; j++) {
        if (arr[j] <= pivot) {
            i++;
            int tmp = arr[i]; arr[i] = arr[j]; arr[j] = tmp;
        }
    }
    int tmp = arr[i + 1]; arr[i + 1] = arr[high]; arr[high] = tmp;
    return i + 1;
}
```

# 자료구조

## 5. 배열 (Array)

### 정의
동일한 타입의 원소들을 **연속된 메모리 공간**에 저장하는 자료구조.  
인덱스를 통해 O(1)로 임의 접근 가능.

### 특징
| 연산 | 시간 복잡도 |
|------|------------|
| 접근 (index) | O(1) |
| 탐색 | O(n) |
| 삽입/삭제 (중간) | O(n) |
| 삽입/삭제 (끝) | O(1) |

### Java 예시
```java
int[] arr = {10, 20, 30, 40, 50};

System.out.println(arr[2]);   // 30 — O(1) 접근

for (int i = 0; i < arr.length; i++) {
    if (arr[i] == 30) System.out.println("found at " + i);
}
```

## 6. 연결 리스트 (Linked List)

### 정의
각 노드가 **데이터**와 **다음 노드를 가리키는 포인터(next)**를 가진 선형 자료구조.  
메모리가 연속적이지 않아도 되며, 삽입/삭제가 O(1).

```
[10 | *] → [20 | *] → [30 | null]
```

### 특징
| 연산 | 시간 복잡도 |
|------|------------|
| 접근 | O(n) |
| 탐색 | O(n) |
| 삽입/삭제 (앞) | O(1) |
| 삽입/삭제 (중간/끝) | O(n) — 탐색 포함 |

### Java 구현
```java
class Node {
    int data;
    Node next;
    Node(int data) { this.data = data; }
}

class LinkedList {
    Node head;

    void addFirst(int data) {
        Node node = new Node(data);
        node.next = head;
        head = node;
    }

    void print() {
        Node cur = head;
        while (cur != null) {
            System.out.print(cur.data + " → ");
            cur = cur.next;
        }
        System.out.println("null");
    }
}
```

## 7. 이중 연결 리스트 (Doubly Linked List)

### 정의
각 노드가 **이전 노드(prev)**와 **다음 노드(next)** 포인터를 모두 가진 연결 리스트.  
양방향 탐색이 가능하고 삭제 연산이 단순 연결 리스트보다 간편하다.

```
null ← [10 | * | *] ↔ [20 | * | *] ↔ [30 | * | *] → null
```

### Java 구현
```java
class DNode {
    int data;
    DNode prev, next;
    DNode(int data) { this.data = data; }
}

class DoublyLinkedList {
    DNode head, tail;

    void addLast(int data) {
        DNode node = new DNode(data);
        if (tail == null) { head = tail = node; return; }
        tail.next = node;
        node.prev = tail;
        tail = node;
    }

    void delete(DNode node) {
        if (node.prev != null) node.prev.next = node.next;
        else head = node.next;
        if (node.next != null) node.next.prev = node.prev;
        else tail = node.prev;
    }
}
```

## 8. 원형 연결 리스트 (Circular Linked List)

### 정의
마지막 노드의 next가 **다시 첫 번째 노드(head)를 가리키는** 연결 리스트.  
순환 구조로, 끝이 없어 순환 큐나 라운드 로빈 스케줄링에 활용된다.

```
→ [10] → [20] → [30] → (head로 되돌아감)
      ↑_________________________↑
```

### Java 구현
```java
class CNode {
    int data;
    CNode next;
    CNode(int data) { this.data = data; }
}

class CircularLinkedList {
    CNode head;

    void add(int data) {
        CNode node = new CNode(data);
        if (head == null) { head = node; node.next = head; return; }
        CNode cur = head;
        while (cur.next != head) cur = cur.next;
        cur.next = node;
        node.next = head;
    }
}
```

## 9. 해시 테이블 (Hash Table)

### 정의
**해시 함수**를 이용해 키(key)를 인덱스로 변환하고, 해당 인덱스에 값을 저장하는 자료구조.  
평균 O(1)로 탐색·삽입·삭제가 가능하다.

### 핵심 개념
- **해시 함수**: key → index 변환 (예: `key % tableSize`)
- **충돌(Collision)**: 서로 다른 키가 같은 인덱스를 가질 때
- **충돌 해결**:
  - **체이닝(Chaining)**: 같은 인덱스에 연결 리스트로 연결
  - **개방 주소법(Open Addressing)**: 빈 슬롯을 탐색해 저장

### 특징
| 연산 | 평균 | 최악 (충돌 많을 때) |
|------|------|---------------------|
| 탐색 | O(1) | O(n) |
| 삽입 | O(1) | O(n) |
| 삭제 | O(1) | O(n) |

### Java 예시 (HashMap)
```java
import java.util.HashMap;

HashMap<String, Integer> map = new HashMap<>();
map.put("apple", 3);
map.put("banana", 5);

System.out.println(map.get("apple"));          // 3
System.out.println(map.containsKey("banana")); // true
map.remove("banana");
```

## 10. 튜플 & 레코드 (Tuple & Record)

### 정의
- **튜플(Tuple)**: 순서가 있는 **불변(immutable)** 값의 묶음. 각 위치에 의미가 있다.
- **레코드(Record)**: 필드명을 가진 데이터 묶음. Java 16+의 `record` 키워드로 불변 클래스를 간결하게 선언 가능.

### 튜플 vs 레코드 비교
| 항목 | 튜플 | 레코드 |
|------|------|--------|
| 접근 방식 | 인덱스 (0, 1, 2...) | 필드명 |
| 가변성 | 불변 | 불변 |
| 의미 전달 | 낮음 | 높음 |

### Java 예시
```java
// Java의 record (Java 16+)
record Point(int x, int y) {}

Point p = new Point(3, 4);
System.out.println(p.x());  // 3
System.out.println(p.y());  // 4

record Pair<A, B>(A first, B second) {}

Pair<String, Integer> pair = new Pair<>("age", 25);
System.out.println(pair.first() + ": " + pair.second()); // age: 25
```

## 11. 트리 (Tree)

### 정의
노드들이 **계층적 부모-자식 관계**로 연결된 비선형 자료구조.  
사이클이 없는 연결 그래프이기도 하다.

### 용어
| 용어 | 설명 |
|------|------|
| 루트(Root) | 최상위 노드 |
| 리프(Leaf) | 자식이 없는 노드 |
| 깊이(Depth) | 루트에서 해당 노드까지의 거리 |
| 높이(Height) | 루트에서 가장 깊은 리프까지의 거리 |
| 차수(Degree) | 자식 노드의 수 |

### 구조 예시
```
        A          ← 루트, 깊이 0
       / \
      B   C        ← 깊이 1
     / \   \
    D   E   F      ← 리프, 깊이 2
```

### 순회 방법
| 방법 | 순서 | 결과 (위 트리 기준) |
|------|------|-------------------|
| 전위(Pre-order) | 루트 → 왼쪽 → 오른쪽 | A B D E C F |
| 중위(In-order) | 왼쪽 → 루트 → 오른쪽 | D B E A C F |
| 후위(Post-order) | 왼쪽 → 오른쪽 → 루트 | D E B F C A |

## 12. 이진 트리 (Binary Tree)

### 정의
각 노드의 자식이 **최대 2개(왼쪽, 오른쪽)**인 트리.

### 종류
| 종류 | 설명 |
|------|------|
| 완전 이진 트리 | 마지막 레벨 제외 모두 채워짐, 왼쪽부터 채워짐 |
| 포화 이진 트리 | 모든 노드가 0 또는 2개의 자식 |
| 이진 탐색 트리(BST) | 왼쪽 < 루트 < 오른쪽 |

### 이진 탐색 트리 (BST)
```
        8
       / \
      3   10
     / \    \
    1   6    14
       / \
      4   7
```
- 탐색: O(log n) 평균, O(n) 최악 (편향)
- 삽입: O(log n) 평균

### Java 구현 (BST 탐색)
```java
TreeNode search(TreeNode root, int key) {
    if (root == null || root.val == key) return root;
    if (key < root.val) return search(root.left, key);
    return search(root.right, key);
}
```

## 13. 힙 (Heap)

### 정의
**완전 이진 트리** 기반의 자료구조로, 부모 노드가 항상 자식보다 크거나(최대 힙) 작다(최소 힙).  
우선순위 큐(Priority Queue) 구현에 사용된다.

### 종류
- **최대 힙(Max Heap)**: 루트가 항상 최댓값
- **최소 힙(Min Heap)**: 루트가 항상 최솟값

### 특징
| 연산 | 시간 복잡도 |
|------|------------|
| 삽입 | O(log n) |
| 최솟값/최댓값 조회 | O(1) |
| 삭제 (루트) | O(log n) |
| 힙 구성 (heapify) | O(n) |

### 최소 힙 구조 예시
```
        1
       / \
      3   5
     / \ / \
    4  8 7  9
```

### Java 예시 (PriorityQueue)
```java
import java.util.PriorityQueue;

PriorityQueue<Integer> minHeap = new PriorityQueue<>();
minHeap.offer(5);
minHeap.offer(1);
minHeap.offer(3);

System.out.println(minHeap.poll()); // 1 (최솟값)
System.out.println(minHeap.poll()); // 3
```

## 14. 그래프 (Graph)

### 정의
**정점(Vertex)**과 **간선(Edge)**으로 이루어진 비선형 자료구조.  
트리는 그래프의 특수한 형태(사이클 없는 연결 그래프).

### 종류
| 종류 | 설명 |
|------|------|
| 방향 그래프 | 간선에 방향 있음 (A → B) |
| 무방향 그래프 | 간선에 방향 없음 (A — B) |
| 가중치 그래프 | 간선에 비용(가중치) 있음 |
| 연결 그래프 | 모든 정점이 연결됨 |

### 표현 방법

**인접 행렬 (Adjacency Matrix)**
```
    0 1 2 3
  0[0,1,0,1]
  1[1,0,1,0]
  2[0,1,0,1]
  3[1,0,1,0]
```
- 공간: O(V²), 간선 조회: O(1)

**인접 리스트 (Adjacency List)**
```
0: [1, 3]
1: [0, 2]
2: [1, 3]
3: [0, 2]
```
- 공간: O(V+E), 간선 조회: O(degree)

### 탐색 방법

**DFS (깊이 우선 탐색)** — 스택/재귀 사용
```java
void dfs(List<List<Integer>> graph, boolean[] visited, int v) {
    visited[v] = true;
    System.out.print(v + " ");
    for (int next : graph.get(v))
        if (!visited[next]) dfs(graph, visited, next);
}
```

**BFS (너비 우선 탐색)** — 큐 사용
```java
void bfs(List<List<Integer>> graph, int start) {
    boolean[] visited = new boolean[graph.size()];
    Queue<Integer> queue = new LinkedList<>();
    queue.offer(start);
    visited[start] = true;
    while (!queue.isEmpty()) {
        int v = queue.poll();
        System.out.print(v + " ");
        for (int next : graph.get(v))
            if (!visited[next]) { visited[next] = true; queue.offer(next); }
    }
}
```

| 자료구조 | 접근 | 탐색 | 삽입 | 삭제 | 특징 |
|----------|------|------|------|------|------|
| 배열 | O(1) | O(n) | O(n) | O(n) | 인덱스 기반 빠른 접근 |
| 연결 리스트 | O(n) | O(n) | O(1) | O(1) | 동적 크기, 앞 삽입 빠름 |
| 이중 연결 리스트 | O(n) | O(n) | O(1) | O(1) | 양방향 탐색 |
| 해시 테이블 | - | O(1)* | O(1)* | O(1)* | 키-값 쌍, 충돌 주의 |
| BST | - | O(log n)* | O(log n)* | O(log n)* | 정렬 유지 |
| 힙 | - | O(n) | O(log n) | O(log n) | 최솟/최댓값 빠른 조회 |
| 그래프 | - | O(V+E) | O(1) | O(E) | 관계 표현 |

## 빌드 도구 (Gradle)

### Gradle이 Java 프로젝트에서 하는 역할

Gradle은 Java 프로젝트의 빌드 자동화 도구로, 소스 코드 컴파일부터 테스트·패키징까지 전 과정을 자동화한다.

### 주요 역할

| 역할 | 설명 |
|------|------|
| 의존성 관리 | `build.gradle`에 선언한 라이브러리를 Maven Central 등에서 자동 다운로드 |
| 컴파일 | 소스 디렉토리의 `.java` 파일을 `.class`(바이트코드)로 변환 |
| 테스트 실행 | 테스트 코드를 컴파일·실행하고 HTML 결과 리포트 생성 |
| 패키징 | 컴파일된 클래스와 리소스를 `.jar` 파일로 묶음 |
| 태스크 의존성 | `build` → `test` → `compileTestJava` → `compileJava` 순으로 자동 실행 |

### 이 프로젝트의 build.gradle 구조

```groovy
plugins { id 'java' }  // Java 플러그인: 컴파일·테스트·패키징 태스크 제공

java {
    toolchain {
        languageVersion = JavaLanguageVersion.of(21)  // JDK 21 사용 지정
    }
}

sourceSets {
    main { java { srcDirs = ['src/main'] } }  // 소스 루트: src/main/
    test { java { srcDirs = ['src/test'] } }  // 테스트 루트: src/test/
}

repositories { mavenCentral() }  // 의존성 다운로드 저장소

dependencies {
    testImplementation 'org.junit.jupiter:junit-jupiter:5.13.4'  // JUnit 5
}

test { useJUnitPlatform() }  // JUnit 5 플랫폼으로 테스트 실행
```

### 주요 Gradle 태스크

| 명령어 | 설명 |
|--------|------|
| `./gradlew build` | 컴파일 + 테스트 + JAR 패키징 전체 실행 |
| `./gradlew test` | 테스트만 실행 (`build/reports/tests/`에 HTML 리포트 생성) |
| `./gradlew compileJava` | 소스 코드 컴파일만 실행 |
| `./gradlew clean` | `build/` 디렉토리 삭제 (빌드 캐시 초기화) |
| `./gradlew dependencies` | 현재 의존성 트리 출력 |

### Gradle 빌드 플로우

```
./gradlew build
     │
     ├── compileJava       →  src/main/  →  build/classes/java/main/
     ├── compileTestJava   →  src/test/  →  build/classes/java/test/
     ├── test              →  JUnit 실행  →  build/reports/tests/test/index.html
     └── jar               →  build/libs/java-algorithm-1.0.0.jar
```

### sourceSets가 필요한 이유

Gradle Java 플러그인의 기본 소스 루트는 `src/main/java/`와 `src/test/java/`이다.  
이 프로젝트는 `src/main/`과 `src/test/`를 직접 소스 루트로 사용하므로 `sourceSets`로 명시해야 한다.

```
기본 구조 (Gradle 기본값)       이 프로젝트 구조
src/main/java/study/Example.java  →  src/main/study/Example.java
src/test/java/study/Day2Test.java →  src/test/study/Day2Test.java
```
