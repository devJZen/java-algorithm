# 자료구조

## 1. 스택 (Stack)

### 정의
**LIFO(Last In, First Out)** 구조. 마지막에 넣은 데이터를 가장 먼저 꺼낸다.  
함수 호출 스택, 괄호 검사, 실행 취소(Undo) 등에 활용된다.

**참고:** [Oracle Java SE 21 — Stack API](https://docs.oracle.com/en/java/javase/21/docs/api/java.base/java/util/Stack.html) · [ArrayDeque API](https://docs.oracle.com/en/java/javase/21/docs/api/java.base/java/util/ArrayDeque.html)

### 특징
| 연산 | 시간 복잡도 |
|------|------------|
| push (삽입) | O(1) |
| pop (삭제) | O(1) |
| peek (조회) | O(1) |
| search | O(n) |

### Java 직접 구현
```java
class Stack<T> {
    private Object[] data;
    private int top;

    Stack(int size) { data = new Object[size]; top = -1; }

    void push(T item) { data[++top] = item; }

    @SuppressWarnings("unchecked")
    T pop() { return (T) data[top--]; }

    @SuppressWarnings("unchecked")
    T peek() { return (T) data[top]; }

    boolean isEmpty() { return top == -1; }
}
```

### Java 표준 라이브러리 (ArrayDeque 권장)
```java
import java.util.ArrayDeque;
import java.util.Deque;

Deque<Integer> stack = new ArrayDeque<>();
stack.push(1);
stack.push(2);
stack.push(3);

System.out.println(stack.peek()); // 3
System.out.println(stack.pop());  // 3
System.out.println(stack.pop());  // 2
```

> `Stack` 클래스는 `Vector`를 상속해 동기화 비용이 있습니다. 단일 스레드 환경에서는 `ArrayDeque` 사용을 권장합니다. ([Oracle Docs 참고](https://docs.oracle.com/javase/tutorial/collections/interfaces/deque.html))

## 2. 큐 (Queue)

### 정의
**FIFO(First In, First Out)** 구조. 먼저 넣은 데이터를 먼저 꺼낸다.  
프린터 대기열, BFS 탐색, 작업 스케줄링에 활용된다.

**참고:** [Oracle Java SE 21 — Queue API](https://docs.oracle.com/en/java/javase/21/docs/api/java.base/java/util/Queue.html) · [Oracle Deque Tutorial](https://docs.oracle.com/javase/tutorial/collections/interfaces/deque.html)

### 특징
| 연산 | 시간 복잡도 |
|------|------------|
| offer / add (삽입) | O(1) |
| poll / remove (삭제) | O(1) |
| peek (조회) | O(1) |

### Java 직접 구현 (원형 큐)
```java
class CircularQueue {
    private int[] data;
    private int front, rear, size;

    CircularQueue(int capacity) {
        data = new int[capacity];
        front = rear = size = 0;
    }

    void offer(int item) {
        data[rear] = item;
        rear = (rear + 1) % data.length;
        size++;
    }

    int poll() {
        int item = data[front];
        front = (front + 1) % data.length;
        size--;
        return item;
    }

    boolean isEmpty() { return size == 0; }
}
```

### Java 표준 라이브러리
```java
import java.util.ArrayDeque;
import java.util.Queue;

Queue<Integer> queue = new ArrayDeque<>();
queue.offer(1);
queue.offer(2);
queue.offer(3);

System.out.println(queue.peek()); // 1
System.out.println(queue.poll()); // 1
System.out.println(queue.poll()); // 2
```

## 3. 덱 (Deque)

### 정의
**Double-Ended Queue**. 앞과 뒤 양쪽에서 삽입과 삭제가 모두 가능한 자료구조.  
스택과 큐의 기능을 모두 갖추고 있어 슬라이딩 윈도우 등에 활용된다.

**참고:** [Oracle Java SE 21 — Deque API](https://docs.oracle.com/en/java/javase/21/docs/api/java.base/java/util/Deque.html) · [Oracle Deque Tutorial](https://docs.oracle.com/javase/tutorial/collections/interfaces/deque.html)

### 특징
| 연산 | 시간 복잡도 |
|------|------------|
| offerFirst / addFirst | O(1) |
| offerLast / addLast | O(1) |
| pollFirst / removeFirst | O(1) |
| pollLast / removeLast | O(1) |
| peekFirst / peekLast | O(1) |

### 덱 vs 스택 vs 큐
| | 스택 | 큐 | 덱 |
|-|------|----|----|
| 삽입 | 뒤(top) | 뒤 | 앞/뒤 |
| 삭제 | 뒤(top) | 앞 | 앞/뒤 |
| 접근 방식 | LIFO | FIFO | 양방향 |

### Java 표준 라이브러리
```java
import java.util.ArrayDeque;
import java.util.Deque;

Deque<Integer> deque = new ArrayDeque<>();
deque.offerFirst(1);  // [1]
deque.offerLast(2);   // [1, 2]
deque.offerFirst(0);  // [0, 1, 2]

System.out.println(deque.peekFirst()); // 0
System.out.println(deque.peekLast());  // 2

deque.pollFirst(); // [1, 2]
deque.pollLast();  // [1]
```

# 정렬 알고리즘

## 4. 병합 정렬 (Merge Sort)

### 정의
배열을 절반으로 재귀적으로 분할한 뒤, 정렬된 두 배열을 **합병(merge)**하는 분할 정복 정렬.  
항상 O(n log n)을 보장하며 안정 정렬이다.

### 특징
| 항목 | 내용 |
|------|------|
| 시간 복잡도 (최선/평균/최악) | O(n log n) |
| 공간 복잡도 | O(n) — 보조 배열 필요 |
| 안정 정렬 | O |

### 동작 과정
```
[5, 3, 1, 4, 2]

분할: [5, 3] | [1, 4, 2]
분할: [5] [3] | [1] [4, 2] → [4] [2]

합병: [3, 5] | [1] [2, 4]
합병: [3, 5] | [1, 2, 4]
합병: [1, 2, 3, 4, 5]
```

### Java 구현
```java
void mergeSort(int[] arr, int left, int right) {
    if (left >= right) return;
    int mid = (left + right) / 2;
    mergeSort(arr, left, mid);
    mergeSort(arr, mid + 1, right);
    merge(arr, left, mid, right);
}

void merge(int[] arr, int left, int mid, int right) {
    int[] tmp = new int[right - left + 1];
    int i = left, j = mid + 1, k = 0;
    while (i <= mid && j <= right)
        tmp[k++] = arr[i] <= arr[j] ? arr[i++] : arr[j++];
    while (i <= mid)   tmp[k++] = arr[i++];
    while (j <= right) tmp[k++] = arr[j++];
    System.arraycopy(tmp, 0, arr, left, tmp.length);
}
```

## 5. 카운팅 정렬 (Counting Sort)

### 정의
각 원소의 **등장 횟수**를 세어 정렬하는 알고리즘.  
비교 기반이 아니어서 O(n + k) 시간 복잡도를 가지며, 값의 범위(k)가 작을 때 매우 빠르다.

### 특징
| 항목 | 내용 |
|------|------|
| 시간 복잡도 | O(n + k) — k: 값의 최대 범위 |
| 공간 복잡도 | O(k) |
| 안정 정렬 | O (누적합 방식 사용 시) |
| 제약 | 정수 또는 이산 값에만 적용 가능 |

### 동작 과정
```
입력: [4, 2, 2, 8, 3, 3, 1]  (최댓값 k = 8)

1. 등장 횟수:
   index: [0, 1, 2, 3, 4, 5, 6, 7, 8]
   count: [0, 1, 2, 2, 1, 0, 0, 0, 1]

2. 인덱스 순서대로 count만큼 출력:
   → [1, 2, 2, 3, 3, 4, 8]
```

### Java 구현
```java
void countingSort(int[] arr, int maxVal) {
    int[] count = new int[maxVal + 1];
    for (int x : arr) count[x]++;

    int idx = 0;
    for (int i = 0; i <= maxVal; i++)
        while (count[i]-- > 0) arr[idx++] = i;
}
```

## 6. 힙 정렬 (Heap Sort)

### 정의
**최대 힙(Max Heap)**을 구성한 후, 루트(최댓값)를 배열 끝으로 옮기는 작업을 반복하는 정렬.  
항상 O(n log n)이며 추가 메모리 O(1)의 제자리 정렬이다.

**참고:** [Oracle Java SE 21 — PriorityQueue API](https://docs.oracle.com/en/java/javase/21/docs/api/java.base/java/util/PriorityQueue.html)

### 특징
| 항목 | 내용 |
|------|------|
| 시간 복잡도 (최선/평균/최악) | O(n log n) |
| 공간 복잡도 | O(1) — 제자리 정렬 |
| 안정 정렬 | X |

### 동작 과정
```
[5, 3, 8, 1, 4]

1. 최대 힙 구성 (heapify): [8, 4, 5, 1, 3]
2. 루트(8)를 끝으로: [3, 4, 5, 1, | 8]  → heapify → [5, 4, 3, 1, | 8]
3. 루트(5)를 끝으로: [1, 4, 3, | 5, 8]  → heapify → [4, 1, 3, | 5, 8]
4. 루트(4)를 끝으로: [3, 1, | 4, 5, 8]  → heapify → [3, 1, | 4, 5, 8]
5. 루트(3)를 끝으로: [1, | 3, 4, 5, 8]
결과: [1, 3, 4, 5, 8]
```

### Java 구현
```java
void heapSort(int[] arr) {
    int n = arr.length;
    for (int i = n / 2 - 1; i >= 0; i--)
        heapify(arr, n, i);
    for (int i = n - 1; i > 0; i--) {
        int tmp = arr[0]; arr[0] = arr[i]; arr[i] = tmp;
        heapify(arr, i, 0);
    }
}

void heapify(int[] arr, int n, int i) {
    int largest = i, left = 2 * i + 1, right = 2 * i + 2;
    if (left  < n && arr[left]  > arr[largest]) largest = left;
    if (right < n && arr[right] > arr[largest]) largest = right;
    if (largest != i) {
        int tmp = arr[i]; arr[i] = arr[largest]; arr[largest] = tmp;
        heapify(arr, n, largest);
    }
}
```
